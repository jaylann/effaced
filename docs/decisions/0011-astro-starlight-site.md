# 0011. Astro Starlight site with griffe API-doc bridge

- **Status:** accepted (supersedes 0005)
- **Date:** 2026-06-11

## Context

ADR 0005 deferred the docs-engine choice because no marketing site was planned, and leaned MkDocs Material for the Python-ecosystem fit. That premise changed: effaced gets a custom-designed marketing page and docs under one bespoke brand (minimal black/white, redaction motif), which is exactly the case where Astro Starlight wins — marketing pages are plain Astro with total design control, docs share the same tokens and components. The cost ADR 0005 flagged (a Python→markdown bridge plus a JS toolchain) is now paid deliberately.

## Decision

- **Astro 5 + Starlight** in `site/` at the repo root — inside the monorepo, outside the uv workspace (`members = ["packages/*"]` does not match it). One repo, two toolchains; `site/` is managed with **pnpm** (pinned via `packageManager` in `site/package.json`).
- **API reference is generated by `scripts/gen_api_docs.py`**, which calls **griffe** directly (no MkDocs anywhere): static analysis of `packages/*/src`, Google docstrings → MDX into `site/src/content/docs/docs/reference/`. Generated files are **gitignored** and rebuilt in CI and by the `just site-*` recipes — docstrings stay the single source of truth, and docstring review *is* API-doc review.
- **Deploy: GitHub Pages now**, built and published from pushes to `stage` (pre-release, stage is the live truth). `astro.config.mjs` reads `SITE_URL`/`BASE_PATH` from the environment, so moving to `effaced.dev` (Cloudflare or Pages + custom domain) later changes only workflow env and the deploy step — no site code.
- **Versioned docs are deferred** (starlight-versions when a 1.0-adjacent release exists; both packages are 0.x today).

## Consequences

- ADR 0005's input discipline carries over unchanged and matters more: ruff `D` Google docstrings render verbatim on the public site.
- The docs build is a separate, path-filtered workflow (`docs.yml`), not a required check; `just ci` stays Python-only.
- The wording rule (mechanisms, never compliance determinations) now binds marketing copy too.
- Dependabot gains an npm ecosystem for `site/`; contributors touching docs need Node + pnpm (`just site-dev`), Python-only contributors are unaffected.

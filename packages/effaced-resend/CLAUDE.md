# effaced-resend

First-party Resend resolver. Depends on `effaced` (workspace source) + `httpx` only — no Resend SDK (it's a thin sync wrapper with no transport seam; direct httpx keeps the test boundary clean).

- `ResendResolver` implements the `Resolver` protocol (structural; `test_resend_conformance.py` runs the shared `effaced.testing.ResolverConformanceSuite`). Keep `name == "resend"` stable forever — it's recorded in audits and outbox entries.
- Refs: `kind="resend"`, `value=<contact email>` — Resend's global-contacts API addresses contacts by email directly (`/contacts/{email}`), so erasure needs no enumeration, no pagination, no audience/segment config. The ref value is itself PII; that's the accepted outbox posture (same as supabase's propagated URLs) — the no-PII pin covers `ResolverError` messages only. The email is sent verbatim, percent-encoded into a single path segment (`quote(value, safe="")` — wrong-target-erasure guard); never normalize or log it.
- **Idempotency:** Resend 404 on get/delete ⇒ absent — export returns empty, erasure reports `already_absent=True`; success, not an error. Transient faults (429, 5xx, connection) propagate for saga retry; other 4xx raise `ResolverError` (taxonomy table in `errors.py`, status-code-only — Resend error bodies are versioned, the status is the stable part).
- Export is `contact.email` (CONTACT), `contact.first_name`/`contact.last_name` (IDENTITY), `contact.unsubscribed` (BEHAVIORAL). `properties` is caller-defined and never exported. The field set lives in `export_records.py` and is behaviour under widened SemVer: removing/recategorising a field is MAJOR, adding is MINOR. Empty-string fields mean "not held" — skipped.
- **Covered surface (issue #123):** `covered_surface` (the `AttestingResolver` capability) returns `RESEND_COVERED_SURFACE` (`covered_surface.py`), built FROM `export_records._STRING_FIELDS` + `_BOOL_FIELDS` — the same tuples `contact_records` iterates to emit — excluding `contact.properties.*` with a reason. The shared suite proves subset + exclusion-absence; `make_fully_populated_resolver` (full contact incl. `unsubscribed`) the enumeration. Declaration of claimed reach, never a compliance determination.
- **No `rectify_subject`, deliberately:** Resend's PATCH cannot change `email` (the one CONTACT field — and the ref value), and the name splits into first/last, so a single category-keyed IDENTITY scalar has no unambiguous target (ADR 0013's anti-Art.16 argument). Don't add it without a field-keyed correction vocabulary.
- Erasure does NOT cover Resend send history (no deletion API) or preserve the unsubscribed flag — documented loudly in README; mechanism, not determination.
- Sync `httpx` wrapped in `asyncio.to_thread`; a fresh client per call — nothing loop- or connection-bound cached on the instance (ADR 0006).
- No live network calls in tests; inject `transport=FakeResendTransport(...)` (`tests/fake_resend_transport.py`) — faking at the httpx transport boundary exercises the real request pipeline.

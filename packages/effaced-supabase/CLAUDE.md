# effaced-supabase

First-party Supabase resolvers. Base depends on `effaced` (workspace source) + `httpx`. Hosts `SupabaseAuthResolver`, `SupabaseStorageResolver`, and `SupabasePostgrestResolver`. `errors.py` is deliberately resolver-agnostic — `raise_for_taxonomy(response, action, *, system=...)` names the rejecting surface in the `ResolverError` message (auth passes `"supabase auth"`, postgrest `"supabase postgrest"`); storage keeps its own inline messages.

- `SupabaseStorageResolver` (`storage_resolver.py`, #57/ADR 0016) rides `effaced-s3`'s S3-compatible machinery via the **optional `storage` extra** (`effaced-s3` + `boto3`); auth-only installs stay httpx-only. It is **not** re-exported from `__init__` (so auth installs import cleanly) — the public import is the deep `from effaced_supabase.storage_resolver import SupabaseStorageResolver`. Module top re-raises `ImportError` pointing at `pip install "effaced-supabase[storage]"` when the extra is absent.
- Keep `name == "supabase_storage"` stable forever (refs: `kind="supabase_storage"`, `value=<key prefix>` e.g. `users/42/`). Construction guard: missing endpoint/key/secret (and no explicit `client=`) → `ConfigurationError`, never a degraded client. SDK retries off, path-style addressing required, fresh boto3 client per call (never cached — ADR 0006).
- **No-versioning erasure:** the gateway has no `ListObjectVersions`; erasure lists current objects (`iter_current_objects`) and `delete_in_batches` them — current-object deletion IS complete erasure. Never call `list_object_versions`. Empty listing → `already_absent=True`; mixed transient per-key failures keep deleting then raise `PartialStorageEraseError` (retryable); all-non-retryable → `ResolverError`. Tests fake at the `S3ObjectClient` boundary (`tests/fake_supabase_storage_client.py`), whose `list_object_versions` raises `NotImplemented` to prove the resolver never touches it.

- `SupabaseAuthResolver` implements the `Resolver` protocol (structural; `test_supabase_auth_conformance.py` runs the shared `effaced.testing.ResolverConformanceSuite`). Keep `name == "supabase_auth"` stable forever — it's recorded in audits and outbox entries.
- Refs: `kind="supabase_auth"`, `value=<gotrue user id>`. Never accept or log emails/phones.
- **Idempotency:** GoTrue 404 on get/delete ⇒ absent — export returns empty, erasure reports `already_absent=True`; success, not an error. Transient faults (429, 5xx, connection) propagate for saga retry; other 4xx raise `ResolverError` (taxonomy table in `errors.py`). The taxonomy keys on status codes only — GoTrue error-body shapes vary across versions.
- Export is `user.email` + `user.phone` (CONTACT) only. `user_metadata`/`app_metadata` are caller-defined and never exported; `identities` is provider-shaped and skipped. The field set lives in `auth_export_records.py` and is behaviour under widened SemVer: removing/recategorising a field is MAJOR. GoTrue's `""` for unset contact fields means "not held" — skipped, not exported.
- **Covered surface (issue #123):** `SupabaseAuthResolver.covered_surface` returns `AUTH_COVERED_SURFACE` (`auth_covered_surface.py`), built FROM `auth_export_records._USER_FIELDS` (no drift), excluding `user.user_metadata*` / `user.app_metadata*` / `user.identities*` with reasons. `SupabaseStorageResolver.covered_surface` returns `STORAGE_COVERED_SURFACE` (`storage_covered_surface.py`), built FROM `effaced_s3.export_records._OBJECT_FIELDS` (storage extra), with the ADR 0016 no-versioning note. Both attest via the `AttestingResolver` capability; the shared suite proves subset + exclusion-absence, and `make_fully_populated_resolver` enumeration. Declaration of claimed reach, never a compliance determination.
- Sync `httpx` wrapped in `asyncio.to_thread`; a fresh client per call — nothing loop- or connection-bound cached on the instance (ADR 0006). Both `Authorization: Bearer` and `apikey` headers carry the service-role key (GoTrue authorizes the former, Supabase's gateway routes on the latter).
- Propagated `httpx.HTTPStatusError` messages embed the request URL (and thus the user id); that lands next to the subject ref in the outbox row, so it's accepted — the no-PII pin covers `ResolverError` messages only (same posture as stripe).
- No live network calls in tests; inject `transport=FakeGoTrueTransport(...)` (`tests/fake_gotrue_transport.py`) — faking at the httpx transport boundary exercises the real request pipeline.

- `SupabasePostgrestResolver` (`postgrest_resolver.py`, #70) reaches a subject's PII in application tables via the PostgREST data API (`/rest/v1/...`); re-exported from `__init__` (httpx-only, no extra). Keep `name == "supabase_postgrest"` stable forever (refs: `kind="supabase_postgrest"`, `value=<subject id>`).
- **Explicit config, no schema discovery:** constructed with `tables: Sequence[PostgrestTable]` (`postgrest_table.py` / `postgrest_column.py` — frozen pydantic; each table names its `subject_column` + PII `columns`). Empty `tables` → `ConfigurationError` (construction guard, like storage). The declared list IS the auditable "which rows/columns does this reach" record.
- Export: per table `GET /rest/v1/{table}?{subject_column}=eq.{id}&select=...` → one `ExportRecord` per **populated** declared column, `source=table.name`, `field=f"{table}.{column}"` (`postgrest_export_records.row_records`); null/missing cells skipped. Erase: per table `DELETE ...` + `Prefer: return=representation`; **empty representation across every table ⇒ `already_absent=True`** (absence is empty-array, NOT 404 — distinct from auth). Adding/recategorising a declared column changes that deployment's export → behaviour under widened SemVer.
- **Injection-safe filtering:** the subject id rides httpx `params` as `eq.<id>` (httpx URL-encodes it) and the table name is `quote(..., safe="")`'d into one path segment — a hostile id matches literally, never widening the filter or hitting a sibling. `test_supabase_postgrest_resolver.py::test_hostile_subject_id_is_matched_literally_without_bleed` + the property tests pin it.
- **Covered surface:** `covered_surface_for(tables)` (`postgrest_covered_surface.py`) builds one `CoveredField` per declared `{table}.{column}` (no fixed exclusions — caller-shaped), with a note that the surface is exactly the declaration (no discovery). Attests via `AttestingResolver`; shared suite proves subset + enumeration (`make_fully_populated_resolver` seeds a row populating every declared column).
- Service-role key bypasses RLS → server-side only; both `Authorization: Bearer` + `apikey` headers carry it, fresh `httpx` client per call (ADR 0006). Tests fake at the transport boundary (`tests/fake_postgrest_transport.py`).
